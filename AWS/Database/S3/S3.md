# Amazon S3 (Simple Storage Service)

Amazon S3 is an object storage service offering industry-leading scalability, data availability, security, and performance.

## 1. Object Storage Mechanics

### Buckets, Keys, and Object Metadata
Data is stored as Objects within Buckets. Each object is identified by a unique Key (the path/filename). Objects also contain Metadata (name-value pairs) detailing content type, size, and custom tags.

### Storage Classes
- **Standard**: For frequently accessed data.
- **Intelligent-Tiering**: Automatically moves data to the most cost-effective access tier based on access frequency.
- **Standard-IA (Infrequent Access)**: For data accessed less frequently but requiring rapid access when needed.
- **One Zone-IA**: Lower cost, stores data in a single AZ.
- **Glacier (Instant, Flexible, Deep Archive)**: For long-term archiving and digital preservation with varying retrieval times.

### S3 Express One Zone
Directory buckets designed for single-digit millisecond latency, ideal for AI, machine learning, and high-performance analytics workloads.

### Data Durability vs. Availability SLAs
S3 provides 99.999999999% (11 9's) of Data Durability (protecting against data loss) and varying Availability SLAs depending on the storage class (e.g., 99.9% for Standard).

## 2. Data Management & Lifecycle Operations

### S3 Lifecycle Policies
Rules to automatically manage your objects over time. Includes Transition rules (moving objects to cheaper storage classes) and Expiration rules (deleting old objects).

### Object Versioning & MFA Delete
- **Object Versioning**: Keeps multiple variants of an object in the same bucket, protecting against accidental overwrites or deletions.
- **MFA Delete**: Requires additional Multi-Factor Authentication to permanently delete an object version or change versioning state.

### S3 Object Lock
Enforces Write Once, Read Many (WORM) compliance policies, preventing objects from being deleted or overwritten for a fixed amount of time or indefinitely.

### S3 Replication
- **Cross-Region Replication (CRR)**: Copies objects to a different AWS Region for disaster recovery or geographic latency reduction.
- **Same-Region Replication (SRR)**: Copies objects to another bucket in the same region, often used for log aggregation.

### Multipart Upload API
Optimizes uploading large objects (greater than 100 MB) by splitting the file into parts, uploading them in parallel, and reassembling them in S3.

## 3. Access Control & Security

### IAM Policies vs. S3 Bucket Policies vs. ACLs
- **IAM Policies**: Attached to users/roles to dictate what S3 actions they can perform.
- **S3 Bucket Policies**: Attached directly to the bucket, defining rules for all objects within (e.g., enforcing HTTPS or cross-account access).
- **Access Control Lists (ACLs)**: Legacy method for granting basic read/write access to individual objects or buckets.

### S3 Access Points
Granular dataset entry points with unique hostnames and access policies, simplifying access management for shared buckets used by multiple different applications or teams.

### Encryption Options
- **SSE-S3**: Server-side encryption with Amazon S3 managed keys.
- **SSE-KMS**: Server-side encryption with AWS KMS managed keys.
- **SSE-C**: Server-side encryption with customer-provided keys.
- **Client-Side Encryption**: Encrypting data before sending it to S3.

### Block Public Access
Account-level and bucket-level settings that act as a safeguard to ensure no buckets or objects are inadvertently made public via ACLs or bucket policies.
