# AWS EKS Basics

Amazon Elastic Kubernetes Service (EKS) is a managed Kubernetes service that makes it easy for you to run Kubernetes on AWS without needing to install, operate, and maintain your own Kubernetes control plane or nodes.

## 1. Core Architecture & Control Plane

### EKS Control Plane
The fully managed, highly available Kubernetes master node infrastructure (running API server, etcd, scheduler) that AWS operates across multiple Availability Zones.

- **How it works**: AWS manages the scaling and availability of the Kubernetes control plane nodes. It automatically detects and replaces unhealthy control plane instances, and it provides automated version upgrades and patching for them.
- **Benefits**: Completely eliminates the operational burden of managing the complex Kubernetes master nodes yourself. You don't have to worry about etcd quorum, backups, or API server high availability.

**CLI Commands (AWS CLI):**
```bash
# Create an EKS cluster (provisions the managed control plane)
aws eks create-cluster \
    --name my-eks-cluster \
    --role-arn arn:aws:iam::123456789012:role/EKSClusterRole \
    --resources-vpc-config subnetIds=subnet-12345,subnet-67890,securityGroupIds=sg-11111

# Describe the cluster to check its status (e.g., CREATING, ACTIVE)
aws eks describe-cluster --name my-eks-cluster
```

### Worker Nodes (Data Plane)
The compute resources (EC2 instances or Fargate pods) that register with the EKS control plane to actually run your containerized applications.

- **Managed Node Groups**: AWS manages the provisioning and lifecycle of the underlying EC2 Auto Scaling groups for you, making updates and scaling easier.
- **Self-Managed Nodes**: You manually create and manage the EC2 instances, install the kubelet, and join them to the cluster.
- **AWS Fargate**: Serverless compute for containers. EKS automatically provisions Fargate infrastructure for your pods based on the resource requests, eliminating the need to manage EC2 nodes entirely.

**CLI Commands (AWS CLI):**
```bash
# Create a managed node group
aws eks create-nodegroup \
    --cluster-name my-eks-cluster \
    --nodegroup-name my-mng-1 \
    --node-role arn:aws:iam::123456789012:role/NodeInstanceRole \
    --subnets subnet-12345 subnet-67890

# List node groups in a cluster
aws eks list-nodegroups --cluster-name my-eks-cluster
```

### kubectl Integration
The standard open-source Kubernetes command-line tool used to interact directly with the EKS cluster API endpoint.

- **How it connects**: `kubectl` relies on an authentication token generated by the AWS CLI. When you run a `kubectl` command, the AWS CLI passes a short-lived IAM-based token to the EKS API server. EKS then maps your IAM identity to a Kubernetes RBAC (Role-Based Access Control) user/group.
- **Usage**: Once your configuration is set up, you use `kubectl` exactly as you would with any standard upstream Kubernetes cluster.

**CLI Commands:**
```bash
# Update your local ~/.kube/config file to connect to the EKS cluster
aws eks update-kubeconfig --region us-east-1 --name my-eks-cluster

# Verify the connection by listing the worker nodes
kubectl get nodes

# View general cluster info and API server endpoint
kubectl cluster-info
```
